# 首页自定义使用说明

## 概述

我已经为你创建了一个自定义首页，包含了你要求的所有功能：
1. ✅ 个人简介
2. ✅ 博客文章目录（三级分类，支持日期归类）
3. ✅ 热点代码仓库
4. ✅ 评论和 Issue 部分

## 文件结构

创建和修改的文件：

```
├── index.html                    # 自定义首页（已创建）
├── _data/
│   ├── repositories.yml          # 代码仓库配置（已创建）
│   └── categories.yml            # 文章分类配置（已创建）
└── _posts/
    └── 示例文章-如何创建博客文章.md  # 示例文章（已创建）
```

## 使用步骤

### 1. 配置代码仓库

编辑 `_data/repositories.yml`，添加你的 GitHub 仓库信息：

```yaml
repositories:
  - name: "仓库名称"
    url: "https://github.com/your-username/repo-name"
    description: "仓库描述"
    language: "Python"  # 或 C++, Java 等
    stars: 128
    forks: 23
    updated: "2024-01-15"
```

### 2. 配置文章分类（可选）

如果需要修改分类结构，编辑 `_data/categories.yml`。

当前已配置的分类：
- **个人日记**：日常生活、随想感悟、读书笔记
- **工作记录**：项目总结、问题解决、团队协作、会议记录
- **技术文章**：算法与数据结构、自动驾驶、机器人学、机器学习与AI、系统设计、工具与教程
- **随想杂谈**：技术趋势、职业发展、生活哲学

### 3. 创建博客文章

在 `_posts` 目录下创建 Markdown 文件，文件命名格式：
```
YYYY-MM-DD-文章标题.md
```

文章前置元数据格式：
```yaml
---
layout: post
title: "文章标题"
date: 2024-01-15 12:00:00 +0800
categories: [分类ID或名称, 子分类ID或名称]
tags: [标签1, 标签2]
repository: https://github.com/your-username/repo  # 可选
---
```

#### 分类 ID 参考

使用分类 ID（推荐）：
- `personal-diary` - 个人日记
- `work-notes` - 工作记录
- `tech-articles` - 技术文章
- `random-thoughts` - 随想杂谈

子分类 ID 示例：
- `daily-life`, `thoughts`, `reading-notes`（个人日记下）
- `project-summary`, `problem-solving`（工作记录下）
- `algorithms`, `autonomous-driving`, `tools-tutorials`（技术文章下）
- `tech-trends`, `career-development`（随想杂谈下）

详见 `_posts/示例文章-如何创建博客文章.md`。

### 4. 配置评论系统（可选）

#### 使用 Giscus（推荐）

1. 访问 https://giscus.app
2. 按照指引配置并获取配置信息
3. 在 `_config.yml` 中添加：

```yaml
comments:
  provider: giscus
  giscus:
    repo: Shen-Kirren/Shen-Kirren.github.io
    repo_id: YOUR_REPO_ID
    category: General
    category_id: YOUR_CATEGORY_ID
    mapping: pathname
    lang: zh-CN
```

#### 使用 Utterances

1. 访问 https://utteranc.es
2. 按照指引配置
3. 在 `_config.yml` 中添加：

```yaml
comments:
  provider: utterances
  utterances:
    repo: Shen-Kirren/Shen-Kirren.github.io
    issue_term: pathname
```

### 5. 自定义个人简介

编辑 `index.html` 文件，找到个人简介部分（第 8-33 行），修改为你自己的内容。

## 文章目录自动生成逻辑

首页会根据以下规则自动生成文章目录：

1. **按分类分组**：根据 `categories.yml` 中的一级分类分组
2. **按子分类分组**：在每个一级分类下，根据子分类进一步分组
3. **按日期分组**：在每个子分类下，按 "YYYY年MM月" 格式分组
4. **显示顺序**：文章按日期倒序排列（最新的在前）

### 示例文章分类展示

假设你有以下文章：
- `2024-01-20-算法学习笔记.md` 分类：`[tech-articles, algorithms]`
- `2024-01-15-工作项目总结.md` 分类：`[work-notes, project-summary]`
- `2024-01-10-日常生活记录.md` 分类：`[personal-diary, daily-life]`

首页会这样展示：

```
📝 技术文章
  └─ 算法与数据结构
      └─ 2024年01月
          └─ 算法学习笔记 (2024-01-20)

💼 工作记录
  └─ 项目总结
      └─ 2024年01月
          └─ 工作项目总结 (2024-01-15)

📖 个人日记
  └─ 日常生活
      └─ 2024年01月
          └─ 日常生活记录 (2024-01-10)
```

## 代码仓库展示

代码仓库会在首页的"热点代码仓库"部分以卡片形式展示，包含：
- 仓库名称（带链接）
- 仓库描述
- 编程语言
- Star 和 Fork 数量
- 最后更新日期

## 样式自定义

首页的所有样式都在 `index.html` 文件的 `<style>` 标签中（第 224-526 行）。

你可以修改：
- 颜色变量（使用 CSS 变量，如 `var(--tb-theme-color)`）
- 布局（Grid 和 Flexbox）
- 字体大小和间距
- 响应式断点

## 注意事项

1. **分类匹配**：文章分类必须与 `categories.yml` 中定义的 ID 匹配，否则可能不会正确显示
2. **日期格式**：文章日期必须符合格式 `YYYY-MM-DD HH:MM:SS +0800`
3. **文件位置**：所有文章必须放在 `_posts` 目录下
4. **文件名格式**：文件名必须以日期开头，格式为 `YYYY-MM-DD-标题.md`
5. **布局类型**：首页使用 `layout: page`，而不是默认的 `layout: home`

## 常见问题

### Q: 文章不显示在首页？
A: 检查：
- 文章是否在 `_posts` 目录下
- 文件名格式是否正确（YYYY-MM-DD-标题.md）
- 分类是否与 `categories.yml` 中的 ID 匹配
- 日期格式是否正确

### Q: 代码仓库不显示？
A: 检查：
- `_data/repositories.yml` 文件格式是否正确
- 缩进是否正确（YAML 对缩进敏感）
- 是否在 `repositories:` 下添加了仓库列表

### Q: 分类没有子分类显示？
A: 检查：
- `categories.yml` 中是否定义了 `subcategories`
- 文章中是否使用了正确的子分类 ID

### Q: 日期分组不正确？
A: 确保文章日期格式为：`2024-01-15 12:00:00 +0800`
