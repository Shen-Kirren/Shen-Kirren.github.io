# Chirpy 主题配置指南

本指南将帮助你配置你的 GitHub Pages 站点，使其成为你想要的样子。

## 一、基础配置（必填项）

### 1. 站点基本信息
打开 `_config.yml` 文件，修改以下关键配置：

#### 基本信息设置
```yaml
# 站点标题
title: Shen-Kirren's Blog  # 改为你的博客标题

# 副标题/标语
tagline: 自动驾驶工程师的技术博客  # 改为你的标语

# 站点描述（用于 SEO）
description: >-
  记录工作心得、分享学习成果、与更多人交流的技术博客。

# 站点 URL（重要！必须填写）
# 格式：https://你的用户名.github.io（注意末尾没有斜杠）
url: "https://Shen-Kirren.github.io"

# 语言设置（改为中文）
lang: zh-CN  # 或保持 en，取决于你是否需要中文界面
```

#### GitHub 和社交媒体信息
```yaml
# GitHub 用户名
github:
  username: Shen-Kirren  # 改为你的 GitHub 用户名

# Twitter（可选，如果不使用可以删除）
twitter:
  username: your_twitter_username

# 个人信息
social:
  name: Shen-Kirren  # 你的全名
  email: your_email@example.com  # 你的邮箱
  links:
    - https://github.com/Shen-Kirren  # GitHub 主页链接
    # - https://twitter.com/your_username  # Twitter（可选）
    # - https://www.linkedin.com/in/your_username  # LinkedIn（可选）
```

### 2. 时区设置
```yaml
# 设置你的时区（中国使用 Asia/Shanghai）
timezone: Asia/Shanghai
```

## 二、个性化配置（可选但推荐）

### 1. 头像设置
```yaml
# 侧边栏头像（支持本地文件或在线图片）
# 本地文件：将图片放在 assets/img/ 目录下，然后填写路径
# 在线图片：直接填写图片的完整 URL
avatar: /assets/img/avatar.jpg  # 或 https://example.com/avatar.jpg
```

### 2. 主题颜色模式
```yaml
# 主题颜色模式
# light - 浅色主题
# dark - 深色主题
# 留空 - 跟随系统设置并允许手动切换
theme_mode:  # 留空表示自动模式（推荐）
```

### 3. 目录（TOC）设置
```yaml
# 是否在文章中显示目录
toc: true  # true 显示，false 不显示
```

### 4. 评论系统配置
Chirpy 主题支持多种评论系统，选择一种配置：

#### Giscus（推荐，使用 GitHub 评论）
```yaml
comments:
  provider: giscus
  giscus:
    repo: Shen-Kirren/Shen-Kirren.github.io  # 仓库名：用户名/仓库名
    repo_id: YOUR_REPO_ID  # 需要到 https://giscus.app 获取
    category: General
    category_id: YOUR_CATEGORY_ID  # 需要到 https://giscus.app 获取
    mapping: pathname
    lang: zh-CN
```

#### Utterances（使用 GitHub Issues）
```yaml
comments:
  provider: utterances
  utterances:
    repo: Shen-Kirren/Shen-Kirren.github.io
    issue_term: pathname
```

### 5. 联系方式配置
编辑 `_data/contact.yml` 文件，填写你的联系方式：

```yaml
- type: github
  icon: "fab fa-github"
  url: https://github.com/Shen-Kirren  # 添加你的 GitHub 链接

- type: email
  icon: "fas fa-envelope"
  url: mailto:your_email@example.com  # 添加邮件链接
  noblank: true

- type: rss
  icon: "fas fa-rss"
  noblank: true
```

## 三、页面内容配置

### 1. 关于页面
编辑 `_tabs/about.md` 文件，替换为你自己的内容：

```markdown
---
icon: fas fa-info-circle
order: 4
---

# 关于我

我是 Shen-Kirren，一名自动驾驶工程师，专注于路径规划与控制（PNC）领域。

## 工作经历

- 曾在百度自动驾驶技术部（ADT）机器人出租车算法组工作 2 年，位于北京
- 目前在 Deeproute.AI（北京）的 AI Safety 组工作

## 博客目的

- 记录我的工作心得
- 分享我的学习成果
- 与更多人交流

## 你可以做什么

- 在这里找到问题的答案
- 了解我学到的东西
- 提出你的建议来完善我的博客
```

### 2. 创建博客文章
在 `_posts` 目录下创建 Markdown 文件，文件名格式：`YYYY-MM-DD-文章标题.md`

示例：
```markdown
---
layout: post
title: "我的第一篇文章"
date: 2024-01-01 12:00:00 +0800
categories: [技术]
tags: [Jekyll, 博客]
---

这里是文章内容...
```

## 四、SEO 和验证设置（可选）

### 1. 社交媒体预览图
```yaml
# 社交媒体分享时显示的预览图
social_preview_image: /assets/img/social-preview.jpg
```

### 2. 网站验证
```yaml
webmaster_verifications:
  google: YOUR_GOOGLE_VERIFICATION_CODE
  baidu: YOUR_BAIDU_VERIFICATION_CODE
```

### 3. 网站分析（可选）
```yaml
analytics:
  google:
    id: YOUR_GOOGLE_ANALYTICS_ID  # Google Analytics
  # 或其他分析服务
```

## 五、高级自定义

### 1. 自定义样式
如果需要自定义 CSS，可以创建以下文件：
- `assets/css/custom.css` - 自定义样式文件

然后在 `_config.yml` 或主题的相关配置中引入。

### 2. 自定义布局
如果需要修改页面布局，可以：
- 创建 `_layouts/` 目录
- 复制主题的布局文件并修改

### 3. 自定义数据
`_data/` 目录用于存放数据文件：
- `contact.yml` - 联系方式
- `share.yml` - 分享配置

## 六、部署前检查清单

在推送到 GitHub 之前，确保：

- [ ] 修改了 `title` 和 `tagline`
- [ ] 填写了正确的 `url`（你的 GitHub Pages 地址）
- [ ] 填写了 `github.username`
- [ ] 填写了 `social.name` 和 `social.email`
- [ ] 更新了 `social.links`
- [ ] 设置了 `timezone`
- [ ] 更新了 `_tabs/about.md` 的内容
- [ ] 配置了 `_data/contact.yml` 的联系方式
- [ ] 测试了本地运行是否正常

## 七、本地测试

在推送之前，建议先本地测试：

```bash
# 安装依赖
bundle install

# 启动本地服务器
bundle exec jekyll serve

# 在浏览器中访问 http://localhost:4000
```

## 八、部署

1. 将所有更改提交到 Git
2. 推送到 GitHub
3. GitHub Actions 会自动构建和部署（如果已配置）
4. 等待几分钟后访问你的站点

## 注意事项

1. **YAML 语法**：注意缩进，使用空格而不是制表符
2. **文件编码**：确保所有文件使用 UTF-8 编码
3. **图片路径**：图片建议放在 `assets/img/` 目录下
4. **配置更改**：修改 `_config.yml` 后需要重启 Jekyll 服务器才能生效
5. **GitHub Pages**：确保仓库设置中已启用 GitHub Pages

祝你配置顺利！如有问题，可以查阅 Chirpy 主题的官方文档。
