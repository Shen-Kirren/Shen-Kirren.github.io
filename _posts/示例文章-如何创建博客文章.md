这是一篇示例文章，展示如何创建博客文章。

## 文章前置元数据（Front Matter）

每篇博客文章都需要在文件开头使用 YAML 格式的前置元数据：

```yaml
---
layout: post           # 布局类型，固定为 post
title: "文章标题"       # 文章标题
date: 2024-01-15 12:00:00 +0800  # 发布日期和时间
categories: [技术文章, 工具与教程]  # 分类（数组格式）
tags: [Jekyll, 博客]    # 标签（数组格式）
repository: https://github.com/username/repo  # 可选：相关代码仓库
---
```

## 分类说明

根据 `_data/categories.yml` 中的配置，你可以使用以下分类：

### 一级分类
1. **个人日记** (`personal-diary`)
2. **工作记录** (`work-notes`)
3. **技术文章** (`tech-articles`)
4. **随想杂谈** (`random-thoughts`)

### 二级分类（子分类）

#### 个人日记的子分类：
- `daily-life` - 日常生活
- `thoughts` - 随想感悟
- `reading-notes` - 读书笔记

#### 工作记录的子分类：
- `project-summary` - 项目总结
- `problem-solving` - 问题解决
- `team-collaboration` - 团队协作
- `meeting-notes` - 会议记录

#### 技术文章的子分类：
- `algorithms` - 算法与数据结构
- `autonomous-driving` - 自动驾驶
- `robotics` - 机器人学
- `ml-ai` - 机器学习与AI
- `system-design` - 系统设计
- `tools-tutorials` - 工具与教程

#### 随想杂谈的子分类：
- `tech-trends` - 技术趋势
- `career-development` - 职业发展
- `life-philosophy` - 生活哲学

### 分类使用示例

```yaml
# 方式 1：使用分类 ID（推荐）
categories: [tech-articles, algorithms]

# 方式 2：使用分类名称（如果分类ID和名称一致）
categories: [技术文章, 算法与数据结构]

# 方式 3：混合使用
categories: [work-notes, 项目总结]
```

## 日期分组

文章会自动按照发布日期进行分组显示，格式为"YYYY年MM月"（例如：2024年01月）。

## 代码仓库链接

如果你在文章前置元数据中添加了 `repository` 字段，文章列表中会显示一个 GitHub 图标，点击可跳转到相关仓库。

## 文章内容

在 YAML 前置元数据之后，使用 Markdown 语法编写文章内容即可。

支持标准的 Markdown 语法：
- 标题（# 到 ######）
- 列表（有序和无序）
- 代码块（```language）
- 引用（>）
- 链接和图片
- 表格
- 等等

## 文章文件命名规则

文件应放在 `_posts` 目录下，命名格式：
```
YYYY-MM-DD-文章标题.md
```

例如：
```
2024-01-15-如何创建博客文章.md
```

**注意**：
- 日期必须符合格式：YYYY-MM-DD
- 标题中的空格和特殊字符可以保留，但建议使用连字符（-）或下划线（_）
- 文件扩展名必须是 `.md` 或 `.markdown`

## 示例文章结构

```markdown
---
layout: post
title: "我的第一篇文章"
date: 2024-01-15 12:00:00 +0800
categories: [tech-articles, algorithms]
tags: [Python, 算法]
repository: https://github.com/username/project
---

# 文章标题

## 章节一

这里是内容...

## 章节二

更多内容...
```

## 提示

1. 确保日期格式正确，否则文章可能不会正确排序
2. 分类和标签会在首页文章列表中显示
3. 代码仓库链接是可选的，但建议技术类文章都添加
4. 文章会自动出现在首页对应分类的日期分组中
